  <action>
    1. Update Cargo.toml platform dependencies:
       - macOS: add `objc2-app-kit = { version = "0.3", features = ["NSWindow", "NSColor", "NSApplication", "NSScreen", "NSView", "NSRunningApplication", "NSGraphicsContext"] }` and `objc2-foundation = "0.3"` under `[target.'cfg(target_os = "macos")'.dependencies]`
       - Windows: add `"Win32_Graphics_Gdi"` to the existing windows-rs features list
       - Linux: no new deps needed (x11rb already present)

    2. In `src/platform/macos.rs`, add a `pub fn show_highlight_border(info: &WindowInfo) -> Result<()>` function:
       - Create 4 NSWindow instances forming a red border frame around the window bounds
       - Border thickness: 4px
       - Each window: NSWindowStyleMask::Borderless, opaque, red backgroundColor, floating level (use raw level 1000 or kCGFloatingWindowLevel), ignoresMouseEvents=true, releasedWhenClosed=false
       - CRITICAL: Convert coordinates from Quartz (top-left origin) to Cocoa (bottom-left origin): `cocoa_y = screen_height - (quartz_y + window_height)`. Use NSScreen::mainScreen().frame().size.height for screen_height.
       - CRITICAL: Must call NSApplication::sharedApplication() and set activationPolicy to .accessory before creating windows. Run NSRunLoop briefly (or use thread::sleep after orderFrontRegardless).
       - The 4 rectangles:
         - Top:    (x, y_cocoa + height - thickness, width, thickness)
         - Bottom: (x, y_cocoa, width, thickness)
         - Left:   (x, y_cocoa + thickness, thickness, height - 2*thickness)
         - Right:  (x + width - thickness, y_cocoa + thickness, thickness, height - 2*thickness)
       - After orderFrontRegardless on all 4, sleep for 3 seconds, then return (windows drop on function exit).

    3. In `src/platform/windows.rs`, add a `pub fn show_highlight_border(info: &WindowInfo) -> Result<()>` function:
       - Create 4 small opaque red windows using CreateWindowExW with WS_EX_TOPMOST | WS_EX_TOOLWINDOW
       - Register a simple window class with red background brush (CreateSolidBrush(RGB(255, 0, 0)))
       - Position the 4 windows as border edges (same 4-rectangle pattern, 4px thickness)
       - ShowWindow with SW_SHOWNA, then sleep 3 seconds, then DestroyWindow all 4

    4. In `src/platform/linux.rs`, add a `pub fn show_highlight_border(info: &WindowInfo) -> Result<()>` function:
       - Connect to X11 display
       - Create 4 X windows with red background (screen.white_pixel replaced with red pixel via alloc_color or hardcoded for TrueColor visuals)
       - Set override_redirect = true, and _NET_WM_STATE_ABOVE + _NET_WM_WINDOW_TYPE_NOTIFICATION hints
       - Map windows, flush, sleep 3 seconds, then destroy and disconnect

    5. In `src/platform/mod.rs`, add re-exports following the existing list_windows pattern:
       ```rust
       #[cfg(target_os = "macos")]
       pub use macos::show_highlight_border;
       #[cfg(target_os = "windows")]
       pub use windows::show_highlight_border;
       #[cfg(target_os = "linux")]
       pub use linux::show_highlight_border;
       ```

    IMPORTANT per research Pitfall 1: macOS coordinate conversion is critical. Pitfall 2: NSApplication must be initialized. Pitfall 3: Windows needs ShowWindow call. Pitfall 4: X11 needs _NET_WM_STATE_ABOVE hint.
