    1. Add `serde = { version = "1.0", features = ["derive"] }` and `serde_json = "1.0"` to [dependencies] in Cargo.toml.

    2. Create `src/json_export.rs` with:

    - `pub fn json_output_path(png_path: &Path) -> PathBuf` -- uses `PathBuf::with_extension("json")`.

    - `#[derive(Serialize)] pub struct WindowInfoJson` with common fields: window_id (u64), title (String), pid (u32), app_name (String), x (i32), y (i32), width (u32), height (u32), platform (String), plus `#[serde(flatten)] pub platform_attrs: serde_json::Map<String, serde_json::Value>` for platform-specific fields.

    - `impl WindowInfoJson { pub fn from_window_info(info: &WindowInfo) -> Self }` -- copies common fields from WindowInfo, sets `platform` to `std::env::consts::OS`, and populates platform_attrs with an empty map (platform modules will provide the attrs via a separate function).

    - `pub fn build_platform_attrs(info: &WindowInfo) -> serde_json::Map<String, serde_json::Value>` -- cfg-gated per platform. For macOS: calls into platform::macos helpers to get window_number, owner_name, owner_pid, sharing_state (for now, populate from WindowInfo fields that already exist: window_id as window_number, app_name as owner_name, pid as owner_pid, sharing_state defaults to 0). For Windows and Linux: similar stubs using available WindowInfo fields. The real platform-specific values will be filled by extending platform modules -- but the JSON structure and serialization must be correct now.

    - `pub fn write_json(info: &WindowInfoJson, path: &Path) -> anyhow::Result<()>` -- creates parent dirs, opens BufWriter, calls `serde_json::to_writer_pretty`.

    3. Add `pub mod json_export;` to `src/lib.rs`.

    Research recommends using `serde_json::Map<String, serde_json::Value>` with `#[serde(flatten)]` instead of a cfg-gated enum, to avoid cross-platform compilation issues (Pitfall 6 in RESEARCH.md).

    Border thickness, overlay duration, and coordinate conversion are NOT part of this task -- those belong to Plan 02.
